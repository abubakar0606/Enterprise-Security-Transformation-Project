## Important Event Fields for SOC Analysis

During an alert investigation, a SOC analyst should review important event fields.

### Agent Name

Identifies the endpoint that generated the security event.

Example:

`dc-01`

### Agent IP Address

Identifies the monitored system IP address.

Example:

`192.168.10.10`

### Timestamp

Shows when the event occurred.

The timestamp helps create an investigation timeline.

### Process Image

Shows the executable associated with process activity.

Example:

`powershell.exe`

PowerShell activity requires context because it can be used for legitimate administration and malicious activity.

### Process ID

Identifies the running process instance.

Process IDs can help correlate related events.

### User

Identifies the account associated with the event.

Example:

`NT AUTHORITY\SYSTEM`

### Target Filename

Shows the file associated with the event.

File information can help identify:

- Script execution
- File creation
- File modification
- Temporary files
- Suspicious file activity

These fields help analysts understand what occurred on the endpoint.

## SOC Alert Triage Process

A SOC analyst should perform initial alert triage before escalating an event.

The following questions should be considered:

1. Which endpoint generated the alert?
2. What time did the event occur?
3. Which Wazuh rule generated the alert?
4. What is the rule severity level?
5. Which user account was involved?
6. Which process generated the activity?
7. Was PowerShell or another scripting engine involved?
8. What file was accessed or created?
9. Is the activity expected on the server?
10. Are there related events around the same timestamp?

The answers help determine whether an event is:

`Benign`

`Suspicious`

or:

`Potentially Malicious`

The event should only be escalated after reviewing available context.

## PowerShell Monitoring from a SOC Perspective

PowerShell is a legitimate Windows administration tool.

However, attackers may also use PowerShell for malicious activity.

Potential malicious uses include:

- Payload execution
- System discovery
- File download
- Credential access
- Security control bypass
- Persistence activity
- Remote command execution

The observed PowerShell event does not automatically confirm malicious activity.

The analyst should review:

- User account
- Parent process
- Command line
- Process path
- Target file
- Event timestamp
- Related alerts
- Expected administrative activity

This demonstrates the importance of contextual alert analysis.
